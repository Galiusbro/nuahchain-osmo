# История комплексного интеграционного теста страховой системы

## Обзор
Этот документ описывает полный жизненный цикл страхового процесса, протестированный в комплексном интеграционном тесте. Тест охватывает взаимодействие всех модулей страховой системы: roles, treasury, policy, premium и claims.

## Участники системы
- **Insurance Authority** - главный администратор системы
- **Insurer** - страховая компания
- **Policy Holder** - держатель полиса (застрахованный)
- **Claims Reviewer** - рецензент претензий
- **Treasury Manager** - менеджер казначейства
- **Oracle** - оракул для внешних данных

## Шаг 1: Настройка ролей
**Цель**: Назначение ролей всем участникам системы

**Действия**:
1. Insurance Authority назначает роль `ROLE_INSURER` страховой компании
2. Назначается роль `ROLE_POLICY_HOLDER` держателю полиса
3. Назначается роль `ROLE_CLAIMS_REVIEWER` рецензенту претензий
4. Назначается роль `ROLE_TREASURY_MANAGER` менеджеру казначейства
5. Назначается роль `ROLE_ORACLE` оракулу

**Проверки**:
- Все роли успешно назначены
- Каждый участник имеет соответствующую роль
- Система ролей функционирует корректно

**Результат**: Все участники получили необходимые права доступа для выполнения своих функций в страховой системе.

**Действия**:

## Шаг 3: Создание полиса
**Цель**: Оформление страхового полиса для клиента

**Действия**:
1. Policy Holder подает заявку на автострахование
2. Устанавливается срок действия полиса: 1 год
3. Определяются атрибуты полиса:
   - Сумма покрытия: 100,000 unuah
   - Франшиза: 1,000 unuah
   - Тип полиса: автострахование
4. Полис привязывается к пулу "pool-001"
5. Включается покрытие: comprehensive, collision

**Детали полиса**:
- Тип: Автострахование
- Срок: 365 дней
- Покрытие: 100,000 unuah
- Франшиза: 1,000 unuah
- Виды покрытия: комплексное и столкновения

**Результат**: Создан действующий страховой полис с уникальным ID и всеми необходимыми параметрами.

## Шаг 4: Создание плана премий
**Цель**: Установка графика платежей по страховому полису

**Действия**:
1. Insurer создает план премий для полиса
2. Устанавливается ежемесячный график платежей
3. Определяется сумма премии: 1,000 unuah в месяц
4. Максимальное количество платежей: 12 (на год)
5. План привязывается к пулу казначейства

**Параметры плана**:
- Тип расписания: ежемесячный
- Период: 30 дней
- Сумма платежа: 1,000 unuah
- Общее количество платежей: 12
- Связанный пул: pool-001

**Результат**: Создан план премий с ежемесячными платежами для поддержания действия полиса.

## Шаг 5: Платежи по премиям
**Цель**: Регистрация платежей держателя полиса

**Действия**:
1. Policy Holder совершает первый платеж: 1,000 unuah
2. Система регистрирует платеж с уникальным ID
3. Policy Holder совершает второй платеж: 1,000 unuah
4. Второй платеж также регистрируется в системе

**Детали платежей**:
- Первый платеж: 1,000 unuah (успешно зарегистрирован)
- Второй платеж: 1,000 unuah (успешно зарегистрирован)
- Общая сумма уплаченных премий: 2,000 unuah

**Результат**: Зарегистрированы два успешных платежа по премиям, полис остается активным.

**Цель**: Профессиональная оценка и принятие решения по претензии

**Действия**:
1. Claims Reviewer проводит тщательное рассмотрение всех доказательств
2. Анализируются фотографии с места аварии
3. Изучается отчет полиции
4. Принимается решение: ОДОБРЕНО
5. Добавляется комментарий: "Претензия одобрена после тщательного рассмотрения доказательств"

**Процесс рассмотрения**:
- Проверка всех предоставленных доказательств
- Соответствие условиям полиса
- Оценка обоснованности суммы ущерба
- Финальное решение: ОДОБРЕНО

**Результат**: Претензия одобрена к выплате после профессионального рассмотрения.

**Цель**: Модификация условий действующего полиса

**Действия**:
1. Insurer инициирует обновление полиса

3. Уменьшается франшиза: с 1,000 до 500 unuah
4. Тип полиса остается: автострахование

**Изменения в полисе**:
- Покрытие: 100,000 → 120,000 unuah (+20%)
- Франшиза: 1,000 → 500 unuah (-50%)
- Улучшенные условия для клиента

**Результат**: Полис успешно обновлен с улучшенными условиями покрытия.

**Детали просрочки**:

- Причина: Неполучение платежа в срок
- Льготный период: истек
- Статус: Требует внимания

**Результат**: Просроченная премия зарегистрирована в системе для дальнейших действий.

4. Oracle теряет соответствующие права доступа

**Детали отзыва**:
- Участник: Oracle
- Отозванная роль: ROLE_ORACLE
- Инициатор: Insurance Authority
- Результат: Роль успешно удалена

**Результат**: Роль Oracle отозвана, права доступа обновлены в соответствии с новыми требованиями.
