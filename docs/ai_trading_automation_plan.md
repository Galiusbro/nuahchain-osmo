# Автотрейдинг с делегированием полномочий

## Цель
Сделать торговлю через `x/assets` максимально удобной для пользователя: один раз подтвердить полномочия, после чего ИИ‑агент торгует автономно, не заставляя пользователя открывать браузер/кошелек для каждой транзакции.

## Выбранная стратегия
1. **Authz + Feegrant**  
   Пользователь один раз подписывает `MsgGrant` (разрешает ИИ исполнять `MsgBuyAsset`/`MsgSellAsset`) и при необходимости `MsgGrantAllowance`, чтобы бот мог оплачивать комиссии из заранее выделенного бюджета. Делегированные лимиты → отзыв токеном/временем.
2. **Сервисный аккаунт ИИ**  
   Отдельный ключ хранится в бекенде/KMS. ИИ подписывает `MsgExec` от своего имени и пересылает в `nuahd` через gRPC. Ключ пользователя остаётся оффлайн.
3. **Слой политик и мониторинга**  
   Перед отправкой транзакции бот проверяет лимиты пользователя (суточный объем, разрешенные символы, диапазон цен) и пишет логи, чтобы пользователь мог аудитить действия.

Такой подход минимизирует fricton для пользователя (один сетап → далее автономно), но сохраняет возможность мгновенно отозвать полномочия.

## Задачи по шагам
1. **UX выдачи полномочий**
   - Реализовать скрипт `scripts/ai_trader_grant.sh` с подсказками и dry-run.
   - Добавить примеры команд в `README.md`.
2. **Конфигурация бота**
   - Создать пакет `services/ai_trader/config`.
   - Написать `config_test.go` с позитивными/негативными кейсами.
3. **Клиент для oracle/assets**
   - Реализовать gRPC клиент и билдер транзакций (подобно `x/assets/client/cli`).
   - Написать интеграционный тест, который запускается против `tests/localnuah`.
4. **Политики и риск-менеджмент**
   - Определить структуру правил (лимиты, стопы, whitelist символов).
   - Покрыть таблицей тестов, включая edge cases (0 лимит, значительные цены).
5. **Мониторинг и аудит**
   - Добавить логгер + экспорт сделок (JSON/REST).
   - Настроить метрики Prometheus (число сделок, ошибки, использованный лимит).
6. **Документация**
   - Описать «быстрый старт» для пользователя и «операции» (отзыв, обновление лимитов).
   - Подготовить чек-лист проверки перед включением бота.

Каждый шаг завершать после прохождения описанных тестов, чтобы пользователь мог безопасно передать управление ИИ.
