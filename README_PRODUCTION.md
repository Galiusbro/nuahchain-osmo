# 🚀 NUAH Blockchain Production Setup

Продакшен-готовые скрипты для безопасной настройки токеномики NUAH блокчейна.

## 📋 Быстрый старт

### 1. Основная настройка
```bash
# Запуск продакшен скрипта
./setup_production_tokenomics.sh

# Следуйте инструкциям на экране
# Скрипт создаст все необходимые файлы и конфигурации
```

### 2. Запуск ноды
```bash
# Запуск в продакшен режиме
./start_production_node.sh

# Мониторинг
./monitor_node.sh
```

### 3. Проверка безопасности
```bash
# Аудит безопасности
./scripts/security_audit.sh

# Проверка токеномики
./check_tokenomics.sh
```

## 📁 Создаваемые файлы

### Ключевые файлы
- `public_keys_registry.json` - Реестр публичных ключей и владельцев
- `public_keys_registry.txt` - Человекочитаемый формат реестра
- `multisig_config.json` - Конфигурация мультисига
- `genesis_backup_TIMESTAMP/` - Резервная копия genesis

### Скрипты
- `start_production_node.sh` - Запуск ноды в продакшене
- `monitor_node.sh` - Мониторинг ноды
- `check_tokenomics.sh` - Проверка токеномики
- `scripts/security_audit.sh` - Аудит безопасности

## 🔐 Безопасность

### Ключевые особенности
- ✅ Использование OS keyring для безопасного хранения ключей
- ✅ Экспорт публичных ключей без приватных данных
- ✅ Безопасная конфигурация сети (отключение небезопасных CORS)
- ✅ Валидация всех конфигураций
- ✅ Автоматическое создание резервных копий
- ✅ Мультисиг конфигурация для критических операций

### Проверки безопасности
```bash
# Полный аудит
./scripts/security_audit.sh

# Проверяет:
# - Keyring backend
# - Конфигурацию безопасности
# - Genesis файл
# - Файловые разрешения
# - Сетевую безопасность
# - Резервные копии
# - Процессы и логи
```

### Проверка распределения
```bash
./check_tokenomics.sh
```

## 🔧 Конфигурация

### Переменные окружения
```bash
export KEYRING_BACKEND=os          # Безопасный keyring
export CHAIN_ID=nuah-1             # ID сети
export MONIKER=nuah-validator      # Имя валидатора
export BINARY=./build/nuahd        # Путь к бинарнику
```

### Порты (по умолчанию)
- **RPC**: 26657 (только localhost)
- **API**: 1317 (только localhost)
- **gRPC**: 9090 (только localhost)
- **P2P**: 26656

## ⚠️ Важные замечания

1. **Keyring Backend**: Всегда используйте `os` для продакшена
2. **Резервные копии**: Регулярно создавайте резервные копии ключей
3. **Мониторинг**: Используйте `monitor_node.sh` для отслеживания состояния
4. **Безопасность**: Запускайте `security_audit.sh` перед продакшеном
5. **Сеть**: Настройте файрвол для ограничения доступа к портам

## 🆘 Поддержка

При возникновении проблем:
1. Проверьте логи: `tail -f nuahd_production.log`
2. Запустите аудит: `./scripts/security_audit.sh`
3. Проверьте статус: `curl -s http://localhost:26657/status`
