# Free Account Module

Модуль `freeaccount` позволяет создавать аккаунты, которые не платят комиссии за транзакции (газ).

## Функциональность

- Создание бесплатных аккаунтов через governance
- Проверка статуса аккаунта (бесплатный или обычный)
- Автоматическое пропускание проверки комиссий для бесплатных аккаунтов

## Использование

### Создание бесплатного аккаунта

Только governance может создавать бесплатные аккаунты:

```bash
# Создание proposal для создания бесплатного аккаунта
nuahd tx gov submit-proposal create-free-account osmo1... \
  --title "Create Free Account" \
  --summary "Create a free account for testing" \
  --from validator \
  --chain-id localnuah \
  --fees 1000uosmo
```

### Проверка статуса аккаунта

```bash
# Проверить, является ли аккаунт бесплатным
nuahd query freeaccount is-free-account osmo1...
```

### Отправка транзакций с бесплатного аккаунта

Транзакции, подписанные бесплатным аккаунтом, автоматически пропускают проверку комиссий:

```bash
# Обычная транзакция без указания fees
nuahd tx bank send osmo1freeaccount... osmo1recipient... 1000uosmo \
  --from freeaccount \
  --chain-id localnuah
  # Без --fees флага!
```

## Архитектура

Модуль состоит из:

1. **Keeper** - управляет состоянием бесплатных аккаунтов
2. **Ante Handler** - модифицированный обработчик комиссий, который пропускает проверку для бесплатных аккаунтов
3. **CLI** - команды для управления и запросов
4. **Types** - определения сообщений и запросов

## Безопасность

- Только governance может создавать бесплатные аккаунты
- Бесплатные аккаунты по-прежнему подвергаются всем остальным проверкам (подпись, nonce, etc.)
- Модуль не влияет на другие аспекты валидации транзакций
